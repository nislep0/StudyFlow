## Мета лабораторної роботи

Метою лабораторної роботи є реалізація інтерактивного прототипу веб‑застосунку **StudyFlow** з використанням **статичних даних**, без підключення до реальної бази даних або бекенд‑API. Прототип має демонструвати основні сценарії використання системи та взаємодію між модулями.

---

## Опис реалізованого прототипу

Інтерактивний прототип реалізовано у frontend‑частині проєкту з використанням **React + TypeScript**. Дані зберігаються в **LocalStorage**, що дозволяє імітувати роботу з персистентним сховищем без реального серверного боку.

Реалізовано такі функціональні модулі:

- Auth (Login / Register / Logout)
- Courses (керування курсами)
- Assignments (керування завданнями)
- Dashboard (агрегація та статистика)

---

## Реалізовані сценарії використання

### 1. Реєстрація користувача (Register)

1. Користувач відкриває сторінку реєстрації.
2. Вводить email, пароль та (опційно) імʼя.
3. Frontend виконує базову валідацію введених даних.
4. Дані користувача зберігаються у LocalStorage.
5. Користувач автоматично авторизується та перенаправляється на Dashboard.

### 2. Авторизація користувача (Login)

1. Користувач відкриває сторінку входу.
2. Вводить email та пароль.
3. Frontend перевіряє наявність користувача у LocalStorage.
4. У разі успіху зберігається стан авторизації.
5. Користувач отримує доступ до захищених сторінок.

### 3. Керування курсами (Courses)

- Створення нового курсу (назва, опис).
- Відображення списку курсів поточного користувача.
- Редагування назви та опису курсу.
- Видалення курсу (разом із повʼязаними завданнями).

Усі зміни зберігаються у LocalStorage.

### 4. Керування завданнями (Assignments)

- Створення завдання з привʼязкою до курсу.
- Поля завдання: назва, дедлайн, пріоритет, статус.
- Зміна статусу завдання (planned / in_progress / done).
- Фільтрація завдань за курсом та статусом.
- Видалення завдань.

### 5. Дашборд (Dashboard)

На дашборді відображається агрегована інформація:

- загальна кількість завдань;
- кількість виконаних завдань;
- кількість завдань у роботі;
- список найближчих дедлайнів (наступні 7 днів).

Дані агрегуються на frontend‑стороні на основі статичного сховища.

## Структура frontend‑частини

```text
src/
  app/            – ініціалізація застосунку та роутинг
  context/        – AuthContext, DataContext
  data/           – робота з LocalStorage та seed‑дані
  features/       – бізнес‑модулі (auth, courses, assignments, dashboard)
  shared/         – спільні компоненти та захищені маршрути
  types/          – доменні типи (User, Course, Assignment)
```

## Використані технології

- React
- TypeScript
- React Router
- LocalStorage (як статичне сховище даних)

## Перевірка роботи прототипу

1. Перейти у директорію `frontend`.
2. Запустити застосунок командою:

```bash
npm run dev
```

3. Відкрити браузер та перевірити сценарії:

- реєстрація та вхід;
- створення курсів і завдань;
- зміна статусів;
- відображення дашборду.
